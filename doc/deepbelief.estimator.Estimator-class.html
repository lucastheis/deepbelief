Class Estimator

source code

This class implements annealed importance sampling for the estimation of partition functions and means to estimate log-probabilities of data samples.

Always estimate the partition function of a deep belief network first.

>>> dbn.estimate_log_partition_function(self, num_ais_samples=100, beta_weights=np.arange(0, 1, 1000))

The choice of the parameters is crucial. More samples and weights will lead to less biased estimates. Only after the partition function has been estimated with appropriate parameters should estimate_log_probability be called.

>>> logprob, lowerbound = dbn.estimate_log_probability(data, num_samples=100)

Taking more samples will reduce the variance of the estimates.

If one of the lower layers is an instance of SemiRBM, then estimate_log_partition_function also has to be run for this layer with a large value for num_ais_samples.

References:

  • Salakhutdinov, R. and Murray, I. (2008). On the Quantitative Analysis of Deep Belief Networks.
  • Theis, L., Gerwinn, S., Sinz, F. Bethge, M. (2010). Likelihood Estimation in Deep Belief Networks.

estimate_log_partition_function(self, num_ais_samples=100, beta_weights=[], layer=-1)

source code 

Estimate the log of the partition function.

beta_weights should be a list of monotonically increasing values ranging from 0 to 1. See Salakhutdinov & Murray (2008) for details on how to set the parameters.

Parameters:
  • num_ais_samples (integer) - number of samples used to estimate the partition function
  • beta_weights (array_like) - annealing weights ranging from zero to one
  • layer (integer) - can be used to estimate the partition function of one of the lower layers
Returns: real
the estimated log partition function

estimate_log_probability(self, X, num_samples=200)

source code 

Estimates the log-probability in nats.

This method returns two values: Optimistic but consistent estimates of the log probability of the given data samples and estimated lower bounds. The parameter num_samples is only relevant for DBNs with at least 2 layers. estimate_log_partition_function() should be run with appropriate parameters beforehand, otherwise the probability estimates will be very poor.

Parameters:
  • X (array_like) - the data points for which to estimate the log-probability
  • num_samples (integer) - the number of Monte Carlo samples used to estimate the unnormalized probability of the data samples
Returns: tuple
a tuple consisting of the estimated log-probabilities (first entry) and estimated lower bounds (second entry)
